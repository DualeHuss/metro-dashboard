About this project

An INST 377 project for the University of Maryland.

The problem

Metro riders in the DMV need a simple way to check train times and station info. WMATA already publishes the data, but it isn't in one easy-to-read dashboard. Riders often have to click around to get a complete picture.

Who it's for

  • UMD students commuting into DC
  • DMV residents and daily Metro riders
  • Tourists trying to figure out the system

How it works

The frontend is plain HTML, CSS, and JavaScript. The backend is a Node.js + Express server that calls the WMATA and OpenWeather APIs (to avoid exposing API keys in the browser). Every search also gets logged into a Supabase Postgres database, which powers the "most searched" chart on the home page.

Data sources

  • WMATA — station list and live arrival predictions
  • OpenWeather — current weather at each station
  • Supabase — stores search history

Technologies used

  • HTML, CSS, JavaScript
  • Node.js and Express (backend)
  • Chart.js (JS library — bar chart on home page)
  • Leaflet (JS library — map on trains page)
  • Supabase (database)
  • Vercel (deployment)
